🛡️ SOC Alert Investigation: Triage cảnh báo IP độc hại

Báo cáo điều tra cảnh báo IP độc hại theo chuẩn SOC Tier 1: context, evidence, decision tree, final classification và recommendation.

SOCSIEMAlert TriageIOCThreat IntelligenceIncident Report

Case Summary

Write-up mô phỏng quy trình SOC Tier 1 khi nhận cảnh báo máy nội bộ kết nối tới IP có reputation xấu. Mục tiêu là không kết luận vội, mà kiểm chứng bằng log, xác định phạm vi ảnh hưởng và đưa ra phân loại cuối cùng.

1. Alert Context

FieldExample valueAnalyst note
Source hostInternal workstationCần xác định user, hostname, asset owner.
Destination IOCExternal IP flagged maliciousCần kiểm tra reputation, ASN, geo, first seen.
DirectionOutboundOutbound tới IOC đáng chú ý hơn inbound scan bị block.
ActionAllowed/BlockedNếu allowed thì rủi ro cao hơn và cần scope kỹ hơn.

2. Investigation Questions

  • IP này có thật sự known malicious ở thời điểm kiểm tra không?
  • Kết nối là một lần hay lặp lại nhiều lần?
  • Có domain/DNS request liên quan trước khi kết nối IP không?
  • Endpoint có process lạ, persistence, scheduled task hoặc file mới tạo không?
  • Có nhiều host nội bộ cùng kết nối tới IOC không?
  • Firewall/Proxy đã allow hay block traffic?

3. Evidence Collection

SIEM alert: chụp rule name, severity, src_ip, dst_ip, timestamp, action.
Threat intelligence: kiểm tra IP reputation từ nhiều nguồn, ghi lại thời điểm kiểm tra.
Endpoint evidence: kiểm tra process/network connection quanh thời điểm alert.
Network evidence: firewall/proxy/DNS log để xác định allow/block và domain liên quan.

Ảnh alert trên SIEM, kết quả reputation, log firewall/proxy và timeline các event liên quan.

5. Final Report Template

Summary:
A SIEM alert reported outbound traffic from an internal host to an IP address with malicious reputation. The alert was reviewed by checking SIEM context, network logs, IOC reputation, and endpoint evidence.

Assessment:
- IOC reputation: suspicious/malicious based on external reputation check.
- Network action: blocked/allowed by firewall.
- Endpoint evidence: no confirmed malicious process observed in current evidence.
- Scope: single host / multiple hosts.

Conclusion:
Classify as [False Positive / Benign True Positive / True Positive / Escalate]. Continue monitoring and escalate if repeated connections or endpoint indicators appear.

6. Lessons Learned

  • Reputation “malicious = yes” chưa đủ để kết luận máy bị nhiễm mã độc.
  • Phải phân biệt event bị block và event đã được allow.
  • Cần timeline theo phút để biết event nào xảy ra trước/sau.
  • Báo cáo tốt phải có evidence, classification và action rõ ràng.
