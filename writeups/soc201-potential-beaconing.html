SOC-201: Potential Beaconing Activity Detected

Bài lab mô phỏng điều tra một alert có dấu hiệu C2 beaconing: host nội bộ kết nối định kỳ ra external IP qua port 443 với kích thước dữ liệu ổn định. Report tập trung vào cách SOC Analyst xác minh pattern, kiểm tra threat intelligence, xác định process và đề xuất containment.

01 · Executive view

Executive Summary

SIEM phát hiện máy trạm HR-WIN-001 với IP nội bộ 192.168.15.2 liên tục kết nối tới external IP 85.208.253.156 qua port 443. Alert đáng chú ý vì kết nối xuất hiện đều mỗi 5 phút và kích thước dữ liệu outbound/inbound gần như cố định.

Pattern này phù hợp với dấu hiệu ban đầu của C2 beaconing. Kết hợp với threat intelligence từ AbuseIPDB và endpoint evidence cho thấy process pwnet.exe tạo kết nối, case nên được xử lý như một security incident có rủi ro cao: isolate host, block destination IP, collect endpoint artifact và escalate sang Tier 2/IR.

Important: Report không kết luận tên malware/family khi chưa có memory/file analysis. Tuy nhiên, bằng chứng hiện có đủ để đánh giá đây là suspicious outbound C2-like behavior và cần containment.

05 · SOC investigation workflow

Investigation Workflow

  1. Validate beaconing pattern: kiểm tra timestamp giữa các connection, xác nhận interval khoảng 5 phút và byte size có lặp lại không.
  2. Confirm source asset: xác định hostname HR-WIN-001, IP nội bộ 192.168.15.2, owner/business unit và mức độ nhạy cảm.
  3. Check destination reputation: tra cứu IP 85.208.253.156 trên AbuseIPDB, VirusTotal, OTX, Talos hoặc nguồn TI nội bộ.
  4. Pivot to endpoint: xác định process, PID, path, hash, parent process, user context và command line tạo kết nối.
  5. Scope the impact: hunt destination IP/domain trên proxy/firewall/DNS để biết có host nội bộ khác beaconing không.
  6. Contain safely: nếu process không hợp lệ hoặc IP có reputation xấu, isolate host và block destination ở firewall/proxy.
  7. Escalate: chuyển Tier 2/IR với evidence package: alert screenshot, network log, process evidence, TI result và timeline.
06 · Analyst assessment

Final Assessment

Verdict: Suspicious / likely true positive C2-like beaconing. Escalation required.

Reason: Alert có periodic interval 5 phút, stable data size, destination IP có lịch sử abuse trong case, và endpoint evidence chỉ ra process pwnet.exe tạo kết nối.

Confidence: High for suspicious beaconing behavior; medium for confirmed malware until file hash, process path, memory capture and persistence evidence are validated.

Business impact: HR workstation có thể chứa dữ liệu nhạy cảm; cần containment nhanh để giảm nguy cơ C2 communication hoặc data exfiltration.

Escalation package: source host, internal IP, destination IP, port, beacon interval, process name/PID, TI result, timestamp discrepancy note.

07 · Response actions

Recommendations & Containment

  • Isolate HR-WIN-001 khỏi network hoặc chuyển sang VLAN quarantine nếu chính sách cho phép.
  • Block outbound connection tới 85.208.253.156 trên firewall/proxy và tạo rule monitoring nếu có destination liên quan.
  • Collect endpoint evidence: process path, file hash, parent process, command line, autoruns/persistence, recent file changes và network connections.
  • Hunt toàn môi trường theo destination IP, process name pwnet.exe, beacon interval tương tự và TLS/HTTPS metadata nếu có.
  • Kiểm tra tài khoản user đăng nhập trên HR-WIN-001; reset credential nếu có dấu hiệu compromise.
  • Preserve evidence trước khi remediation: screenshot alert, export raw logs, collect sample nếu chính sách cho phép.
08 · Detection improvement

Limitations & Next Improvements

  • Bổ sung DNS/proxy/firewall log để biết process kết nối tới IP trực tiếp hay qua domain.
  • Thêm EDR/Sysmon telemetry để có command line, parent process, file hash và persistence location.
  • Viết correlation rule: periodic outbound + low reputation destination + uncommon process + sensitive asset = High severity.
  • Thêm baseline theo ứng dụng hợp lệ để tránh nhầm với update agent hoặc health-check service.
  • Tạo hunt query cho toàn hệ thống: destination.ip = 85.208.253.156 OR process.name = pwnet.exe.
