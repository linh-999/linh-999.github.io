⚙️ Research Notes: Core IP, Streaming Telemetry & Network Automation

Tổng hợp hướng nghiên cứu về vận hành Core IP/Truyền dẫn, thu thập telemetry, tự động hóa kiểm tra cấu hình và hỗ trợ NOC/SOC.

Network OperationsAutomationTelemetryAIOps

Mục tiêu

  • Xây dựng hướng nghiên cứu phù hợp vị trí Network Operations/Core IP.
  • Hiểu vì sao automation giúp giảm thao tác thủ công trong vận hành mạng.
  • Đề xuất lab nhỏ để kiểm tra trạng thái thiết bị, cấu hình và cảnh báo bất thường.

Vấn đề trong vận hành mạng

  • Hệ thống Core IP/Truyền dẫn có nhiều thiết bị, link và dịch vụ.
  • Nếu chỉ kiểm tra thủ công bằng CLI, kỹ sư mất nhiều thời gian và dễ bỏ sót lỗi.
  • Sự cố mạng cần phát hiện nhanh: link down, CPU cao, interface error, route thay đổi hoặc mất neighbor.

Ý tưởng giải pháp

  • Dùng script để thu thập thông tin thiết bị định kỳ.
  • Chuẩn hóa template cấu hình để giảm sai sót.
  • Dùng telemetry để lấy dữ liệu trạng thái gần real-time.
  • Tạo dashboard hoặc báo cáo giúp NOC theo dõi sức khỏe mạng.

Lab đề xuất

  • Packet Tracer/GNS3/EVE-NG dựng mô hình router-switch cơ bản.
  • Python Netmiko/NAPALM kết nối thiết bị và lấy show command.
  • So sánh running-config với baseline.
  • Gửi cảnh báo khi interface down hoặc route biến mất.

Hướng phát triển

  • Kết hợp AI assistant để tóm tắt log và đề xuất bước kiểm tra.
  • Tự động tạo checklist troubleshooting theo loại sự cố.
  • Tích hợp với ticket/SOAR để quản lý quy trình xử lý.

Kỹ năng rút ra

  • Có tư duy vận hành mạng theo hướng tự động hóa.
  • Biết liên kết kiến thức CCNA với automation và monitoring.
  • Phù hợp portfolio cho Network Operations, NOC, SOC hạ tầng.

Command / Query tham khảo

show interfaces status
show ip route
show ip ospf neighbor

Ghi chú: Nội dung này được viết theo hướng portfolio sinh viên/thực tập, có thể bổ sung ảnh lab, topology, screenshot dashboard hoặc log evidence sau khi có dữ liệu thực tế.
