1. Lab Objective

Mục tiêu của lab là ghi lại cách triển khai, kiểm thử và phân tích evidence theo hướng SOC Analyst. Bài viết không chỉ lưu command/cấu hình, mà phải trả lời: use case phát hiện gì, log đi qua đâu, evidence nào chứng minh alert và hạn chế hiện tại là gì.

2. Use Case & Log Flow

Endpoint / Traffic→Agent / Sensor→Manager / SIEM→Dashboard→Analyst Report
  • Network device sinh metric/log về interface, route và health.
  • Collector nhận dữ liệu định kỳ hoặc streaming.
  • Storage/indexer lưu metric theo time-series.
  • Dashboard hiển thị trạng thái và trend.
  • Automation script tạo health-check report hoặc cảnh báo.

3. Configuration & Evidence Checklist

Evidence checklist: ghi lại cấu hình agent/rule, dashboard hoặc alert detail khi có kết quả lab. Tên file evidence nên thể hiện rõ nội dung như wazuh-rule-file-integrity.png hoặc kibana-portscan-dashboard.png.

4. Detection & Analysis

  • Interface flap: trạng thái link đổi liên tục trong thời gian ngắn.
  • High error/discard: counter lỗi tăng bất thường.
  • Latency/loss spike: ảnh hưởng chất lượng dịch vụ.
  • Routing change: thay đổi route/neighbor cần kiểm tra.

5. Conclusion

Lab liên quan trực tiếp đến project Network Operations & Streaming Telemetry Lab. Kết luận cần ghi rõ rule phát hiện được gì, còn thiếu dữ liệu nào và action nào phù hợp nếu đây là môi trường thật.

  • Chuẩn hóa checklist health-check theo thiết bị/dịch vụ.
  • Lưu baseline để phát hiện deviation.
  • Tạo báo cáo định kỳ cho link/device critical.
