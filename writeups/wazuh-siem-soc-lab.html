1. Lab Objective

Mục tiêu của lab là ghi lại cách triển khai, kiểm thử và phân tích evidence theo hướng SOC Analyst. Bài viết không chỉ lưu command/cấu hình, mà phải trả lời: use case phát hiện gì, log đi qua đâu, evidence nào chứng minh alert và hạn chế hiện tại là gì.

2. Use Case & Log Flow

Endpoint / Traffic→Agent / Sensor→Manager / SIEM→Dashboard→Analyst Report
  • Wazuh agent cài trên Linux endpoint.
  • Agent thu thập auth log, FIM event và command/audit log.
  • Wazuh manager decode, rule match và gán severity.
  • Indexer lưu dữ liệu phục vụ search và dashboard.
  • Dashboard hiển thị alert để analyst triage.

3. Configuration & Evidence Checklist

Evidence checklist: ghi lại cấu hình agent/rule, dashboard hoặc alert detail khi có kết quả lab. Tên file evidence nên thể hiện rõ nội dung như wazuh-rule-file-integrity.png hoặc kibana-portscan-dashboard.png.

4. Detection & Analysis

  • FIM: thay đổi trong /etc/passwd, /etc/shadow hoặc thư mục cấu hình quan trọng.
  • Privileged command: sudo/su command bất thường ngoài baseline.
  • Authentication: nhiều failed login từ cùng source hoặc cùng username.
  • Suspicious process: command có dấu hiệu download-execute hoặc encoded payload.

5. Conclusion

Lab liên quan trực tiếp đến project Wazuh SOC Use Case Pack. Kết luận cần ghi rõ rule phát hiện được gì, còn thiếu dữ liệu nào và action nào phù hợp nếu đây là môi trường thật.

  • Định nghĩa danh sách file critical và user/service account hợp lệ.
  • Tách severity theo asset criticality.
  • Viết playbook riêng cho FIM, auth failure và suspicious command.
  • Thêm enrichment từ CMDB hoặc owner của asset.
