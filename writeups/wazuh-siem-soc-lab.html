📡 Wazuh SIEM/SOC Lab: Manager, Agent và Detection Flow

Ghi chú dựng lab Wazuh để hiểu kiến trúc SIEM/SOC, luồng log từ agent về manager, rule, dashboard và demo phát hiện hành vi.

WazuhSIEMBlue TeamLab

Mục tiêu

  • Hiểu vai trò của Wazuh Manager, Agent, Indexer và Dashboard.
  • Biết cách agent thu thập log endpoint và gửi về server để phân tích.
  • Chuẩn bị nền tảng cho các demo như file integrity, sudo privilege escalation, USB event hoặc brute force SSH.

Kiến trúc tổng quan

  • Wazuh Agent được cài trên endpoint Linux/Windows để thu thập log hệ thống, audit log, security event và file integrity event.
  • Wazuh Manager nhận log, áp rule, sinh alert và chuyển dữ liệu sang indexer.
  • Dashboard giúp SOC Analyst tìm kiếm, lọc alert, xem rule ID, severity, MITRE mapping và host liên quan.

Use case demo đề xuất

  • File Integrity Monitoring: tạo/sửa/xóa file trong thư mục được giám sát.
  • Linux Privilege Escalation: kiểm tra event liên quan sudo hoặc thay đổi quyền.
  • SSH Brute Force: nhiều lần đăng nhập sai từ cùng một IP.
  • Windows Security Event: theo dõi login failure, user creation hoặc policy change.

Báo cáo lab mẫu: File Integrity Monitoring bằng Wazuh

Phần này có thể dùng trực tiếp làm nội dung write-up cho lab Wazuh. Khi có ảnh thật, chỉ cần thay các ô bằng chứng bằng screenshot của bạn.

1. Mục tiêu use case

Use case này mô phỏng tình huống SOC cần phát hiện thay đổi bất thường trên endpoint. Wazuh Agent được cấu hình theo dõi một thư mục quan trọng. Khi người dùng tạo, sửa hoặc xóa file trong thư mục đó, agent sẽ thu thập sự kiện, gửi về Wazuh Manager để đối chiếu rule và hiển thị alert trên Dashboard.

  • Phát hiện hành vi tạo mới file trong thư mục được giám sát.
  • Phát hiện hành vi chỉnh sửa nội dung file.
  • Phát hiện hành vi xóa file hoặc thay đổi metadata.
  • Ghi nhận thông tin endpoint, đường dẫn file, thời gian xảy ra sự kiện và mức độ cảnh báo.

2. Sơ đồ luồng log

EndpointMáy Linux/Windows phát sinh sự kiện
AgentThu thập log và FIM event
ManagerPhân tích, áp rule, tạo alert
IndexerLưu trữ và lập chỉ mục dữ liệu
DashboardSOC xem alert và điều tra

Luồng xử lý chính: Endpoint phát sinh sự kiện thay đổi file → Wazuh Agent đọc cấu hình giám sát → Agent gửi event về Manager → Manager so khớp rule → alert được lưu vào Indexer → Dashboard hiển thị cho SOC Analyst kiểm tra.

3. Cấu hình agent/rule

Trên endpoint, cấu hình File Integrity Monitoring có thể đặt trong file ossec.conf. Ví dụ bên dưới theo dõi thư mục lab /home/khanhlinh/fim-lab.

<syscheck>
  <frequency>300</frequency>
  <directories check_all="yes" realtime="yes">/home/khanhlinh/fim-lab</directories>
</syscheck>
Ảnh cần chèn 1: Cấu hình agent

Chèn ảnh file ossec.conf hoặc màn hình terminal thể hiện thư mục đang được Wazuh Agent giám sát.

Ảnh cần chèn 2: Trạng thái agent

Chèn ảnh agent đã kết nối thành công với manager, ví dụ màn hình agent_control -l hoặc trang Agents trên Wazuh Dashboard.

4. Các bước thực hiện lab

  1. Tạo thư mục lab trên endpoint: mkdir -p /home/khanhlinh/fim-lab.
  2. Cấu hình Wazuh Agent giám sát thư mục này.
  3. Khởi động lại agent để nhận cấu hình mới: sudo systemctl restart wazuh-agent.
  4. Tạo file kiểm thử: echo "test fim" > /home/khanhlinh/fim-lab/test.txt.
  5. Sửa nội dung file: echo "update content" >> /home/khanhlinh/fim-lab/test.txt.
  6. Xóa file: rm /home/khanhlinh/fim-lab/test.txt.
  7. Vào Wazuh Dashboard để kiểm tra alert liên quan đến File Integrity Monitoring.

5. Alert trên dashboard

Khi rule hoạt động đúng, Dashboard sẽ hiển thị alert liên quan đến thay đổi file. SOC Analyst cần kiểm tra các trường quan trọng như agent name, rule ID, severity, file path, event type và timestamp.

Trường thông tin Ý nghĩa khi điều tra
Agent / Hostname Xác định endpoint nào phát sinh sự kiện.
Rule ID / Rule Description Cho biết rule nào đã match và mô tả hành vi được phát hiện.
Severity Level Ước lượng mức độ ưu tiên cần xử lý.
File Path Xác định file hoặc thư mục bị tạo, sửa, xóa.
Timestamp Xác định thời điểm xảy ra sự kiện để đối chiếu log khác.
Ảnh cần chèn 3: Alert trên Wazuh Dashboard

Chèn ảnh màn hình alert FIM, trong đó nhìn rõ agent, rule description, level, file path và thời gian phát hiện.

6. Kết quả phân tích

Sau khi kiểm tra alert, có thể kết luận Wazuh đã ghi nhận được sự kiện thay đổi file trong thư mục được giám sát. Alert có đủ thông tin để SOC Analyst biết máy nào phát sinh sự kiện, file nào bị tác động và hành vi thuộc loại tạo mới, chỉnh sửa hay xóa.

Phát hiện được: thay đổi file trong thư mục quan trọng, bao gồm tạo mới, sửa nội dung và xóa file.
Bằng chứng chính: alert trên dashboard, rule ID, đường dẫn file, agent name và timestamp.
Ý nghĩa SOC: hỗ trợ phát hiện thay đổi trái phép trên endpoint, đặc biệt ở thư mục cấu hình, thư mục web hoặc thư mục chứa dữ liệu nhạy cảm.

7. Hạn chế của use case

  • Rule chỉ xác nhận có thay đổi file, chưa tự kết luận hành vi là độc hại hay hợp lệ.
  • Nếu thư mục giám sát quá rộng, hệ thống có thể sinh nhiều alert gây nhiễu.
  • Cần whitelist hoặc tuning cho các tiến trình hợp lệ như update phần mềm, backup hoặc deployment.
  • Để đánh giá chính xác hơn, cần đối chiếu thêm user login, process execution, command history và network connection.

8. Kết luận báo cáo

Lab File Integrity Monitoring cho thấy Wazuh có thể phát hiện và ghi nhận các thay đổi trên endpoint thông qua agent. Luồng log từ Endpoint → Agent → Manager → Indexer → Dashboard hoạt động đúng, alert hiển thị đầy đủ thông tin phục vụ quá trình điều tra cơ bản. Tuy nhiên, để triển khai thực tế trong SOC, rule cần được tuning để giảm false positive và cần kết hợp thêm log xác thực, tiến trình và network để đánh giá mức độ rủi ro chính xác hơn.

Kỹ năng rút ra

  • Nắm được kiến trúc SIEM cơ bản.
  • Biết đọc alert theo rule ID, severity, agent, source log và MITRE technique.
  • Có nền tảng để so sánh Wazuh với ELK/Splunk ở mức sinh viên/thực tập.

Command / Query tham khảo

sudo systemctl status wazuh-agent
sudo tail -f /var/ossec/logs/ossec.log
agent_control -l
