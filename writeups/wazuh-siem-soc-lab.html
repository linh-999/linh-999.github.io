📡 Wazuh SIEM/SOC Lab: Manager, Agent và Detection Flow

Báo cáo lab Wazuh theo chuẩn SOC: kiến trúc, use case, rule logic, dashboard evidence, kết quả detection và hạn chế.

WazuhSIEMBlue TeamFIMSSH Brute ForceSOC Lab

Lab Report Summary

Lab dựng Wazuh để hiểu cách một nền tảng SIEM/SOC thu thập log endpoint, phân tích rule, hiển thị alert và hỗ trợ analyst điều tra. Nội dung được viết theo format báo cáo lab chuyên nghiệp: mục tiêu, kiến trúc, use case, bước triển khai, bằng chứng, kết quả, hạn chế và hướng cải thiện.

DateNov 2025
PlatformWazuh Manager, Agent, Indexer, Dashboard
RoleSOC Analyst Lab Builder
Use casesFIM, sudo, SSH brute force, USB event

1. Objective

  • Hiểu vị trí của Wazuh Agent, Manager, Indexer và Dashboard trong kiến trúc SOC.
  • Triển khai luồng log từ endpoint về dashboard để analyst quan sát alert.
  • Demo một số use case cơ bản và viết kết luận rule phát hiện được gì.
  • Nhận diện hạn chế: rule noise, thiếu context, cần tuning.

3. Use Case Design

Use caseTriggerDetection logicSOC interpretation
File Integrity MonitoringFile nhạy cảm bị tạo/sửa/xóa.Agent theo dõi path đã cấu hình và tạo event khi hash/metadata thay đổi.Có thể là admin change hợp lệ hoặc dấu hiệu tampering.
Sudo privilege escalationUser chạy lệnh sudo.Linux auth log có sudo command, rule match theo event type.Cần kiểm tra user, lệnh được chạy, thời điểm.
SSH brute forceNhiều failed login liên tục.Nhiều event authentication failure trong time window.Nếu từ external IP cần block/monitor, nếu lab thì tag test traffic.
USB eventThiết bị removable được gắn vào endpoint.OS log ghi nhận device connection.Cần policy: USB có được phép không, user nào sử dụng.

4. Lab Procedure

  1. Cài Wazuh Manager/Indexer/Dashboard hoặc dùng deployment có sẵn.
  2. Cài Wazuh Agent trên endpoint Linux/Windows.
  3. Kiểm tra agent connected trên dashboard.
  4. Cấu hình use case: FIM path hoặc log source cần theo dõi.
  5. Tạo hành vi test trong lab: sửa file, login sai SSH, chạy sudo, gắn USB.
  6. Filter dashboard theo agent.name, rule.id, timestamp để lấy bằng chứng.
  7. Ghi lại kết quả: alert có xuất hiện không, có đủ context không, false positive ở đâu.

7. Conclusion

Wazuh phù hợp để học SOC vì thể hiện đầy đủ pipeline: endpoint telemetry → rule matching → indexed alert → dashboard triage. Tuy nhiên, alert chỉ là tín hiệu ban đầu. Analyst vẫn cần kiểm tra context, asset, user, baseline và policy trước khi kết luận. Lab nên được nâng cấp bằng cách thêm playbook, correlation với network log và rule tuning theo môi trường.
