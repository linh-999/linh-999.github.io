Wazuh SIEM/SOC Lab Report

Bài lab trình bày cách triển khai luồng giám sát Wazuh theo tư duy SOC: endpoint sinh log, agent thu thập, manager xử lý rule, indexer lưu trữ và dashboard hỗ trợ analyst triage. Nội dung tập trung vào cách viết use case, đọc alert và đánh giá limitation.

01 · Notes

Lab objective

  • Thiết kế luồng log Endpoint → Agent → Manager → Indexer → Dashboard.
  • Theo dõi File Integrity Monitoring, authentication failure và command activity.
  • Viết checklist triage cho alert Wazuh ở mức SOC L1.
02 · Notes

Log flow

  • Endpoint sinh log hệ thống và hoạt động bảo mật.
  • Wazuh agent thu thập log và gửi về manager.
  • Manager áp dụng decoder/rule để tạo alert.
  • Indexer lưu trữ dữ liệu để dashboard truy vấn và analyst review.
03 · Notes

Detection use cases

  • FIM: phát hiện file quan trọng bị sửa, xóa hoặc tạo mới.
  • Authentication: phát hiện nhiều lần đăng nhập thất bại hoặc đăng nhập bất thường.
  • Privileged activity: ghi nhận lệnh đặc quyền và hành vi admin cần kiểm tra.
04 · Notes

Analyst conclusion

  • Lab giúp hiểu vai trò của từng thành phần trong Wazuh.
  • Rule chỉ có giá trị khi analyst biết đọc raw log, context và false-positive condition.
  • Cần bổ sung asset inventory, user context và maintenance window để vận hành tốt hơn.
