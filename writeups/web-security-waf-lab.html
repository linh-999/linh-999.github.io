🌐 Web Security & WAF Lab: OWASP Top 10 Detection Notes

Ghi chú kiểm thử web và phân tích log phòng thủ: payload, WAF action, web access log và khuyến nghị mitigation.

Web SecurityOWASPBurp SuiteWAFLog Analysis

Lab Summary

Write-up ghi lại cách kiểm thử web cơ bản và chuyển kết quả kiểm thử thành góc nhìn phòng thủ. Nội dung tập trung vào cách SOC Analyst đọc WAF/web log, không chỉ mô tả payload.

DateApr 2025
FocusOWASP Top 10 detection
ToolsBurp Suite, ZAP, WAF logs
OutputDetection notes + mitigation

1. Analyst Approach

  1. Gửi request test qua proxy để quan sát payload.
  2. Ghi lại endpoint, parameter, method và response.
  3. Kiểm tra WAF có block, log hay allow payload.
  4. Đối chiếu với web access log để biết request có tới server không.
  5. Viết kết luận: exploit thành công, bị block, hay chỉ là scanner noise.

2. Evidence Table

TestEvidenceDefensive interpretation
SQLi string in parameterWAF rule ID, 403 response, request sample.Rule hoạt động, cần xác minh backend không xử lý payload.
XSS payloadEncoded payload, reflected response or block page.Nếu reflect thì cần output encoding; nếu block thì monitor repeated attempts.
Login brute forceRepeated POST, failed status, same source IP.Cần rate limit, account lockout hoặc MFA.

3. Conclusion

Web security lab tốt nên có 2 góc nhìn: attacker gửi payload như thế nào và defender nhìn thấy gì trong log. Khi viết portfolio, phần quan trọng nhất là chứng minh được mình biết biến request/response thành finding có severity, evidence và recommendation.
