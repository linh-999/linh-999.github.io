Web Security & WAF Analysis Notes

Write-up ghi lại cách phân tích một WAF alert: đọc rule ID, payload, endpoint, HTTP method, response code, user-agent và đối chiếu với web access log để xác định request bị block hay có impact lên backend.

01 · Notes

Analysis focus

  • Xác định request có phải exploit attempt hay test hợp lệ.
  • Kiểm tra payload và parameter bị rule match.
  • Đối chiếu WAF action với response code từ backend.
02 · Notes

Evidence fields

  • Source IP, method, URI, parameter, payload, rule ID, action, status code.
  • User-agent và request rate để phân biệt scanner/tool với browser bình thường.
  • Endpoint criticality để đánh giá impact.
03 · Notes

False-positive handling

  • Xác nhận với app owner khi parameter hợp lệ bị WAF block.
  • Tune rule theo endpoint/parameter cụ thể, không whitelist quá rộng.
  • Ghi maintenance hoặc scanner nội bộ vào allowlist có thời hạn.
04 · Notes

Conclusion

  • WAF alert cần được đọc cùng access log và app context.
  • Mitigation tốt phải giữ cân bằng giữa giảm noise và không làm yếu khả năng phòng thủ.
