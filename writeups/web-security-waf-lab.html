🌐 Web Security & WAF Lab: OWASP Top 10 Detection Notes

Ghi chú kiểm thử web cơ bản với Burp Suite/OWASP ZAP, phân tích request-response, nhận diện lỗi phổ biến và đề xuất rule WAF.

Web SecurityOWASPBurp SuiteWAF

Mục tiêu

  • Luyện tư duy kiểm thử web application theo OWASP Top 10.
  • Biết dùng proxy để quan sát request/response.
  • Hiểu cách WAF phát hiện payload bất thường và ghi log.

Môi trường lab

  • Ứng dụng web lab như DVWA/Juice Shop hoặc web test nội bộ.
  • Burp Suite/OWASP ZAP để intercept request.
  • WAF hoặc rule mô phỏng để chặn payload cơ bản.
  • Web server log để đối chiếu request bị chặn hoặc cho phép.

Các lỗi kiểm thử

  • Broken Access Control: thử truy cập tài nguyên không thuộc quyền user.
  • Injection: kiểm tra input có bị nối trực tiếp vào query/command không.
  • Authentication weakness: kiểm tra login failure, brute force protection và session handling.
  • Security Misconfiguration: header thiếu, directory listing, debug mode hoặc default credential.

Cách phân tích log

  • Xác định URI, method, status code, source IP và user-agent.
  • So sánh request bình thường và request có payload.
  • Kiểm tra WAF action: allow, block, challenge hoặc log only.
  • Ghi lại rule/signature nào match payload.

Kết luận mẫu

  • Lab giúp hiểu mối quan hệ giữa tấn công web, log web server và log WAF.
  • Khi đưa vào môi trường thật cần giảm false positive bằng cách tuning rule theo endpoint, parameter và business logic.

Kỹ năng rút ra

  • Biết đọc HTTP request/response.
  • Biết viết báo cáo vulnerability ngắn.
  • Hiểu cách phòng thủ ở tầng WAF và tầng ứng dụng.

Command / Query tham khảo

GET /login.php HTTP/1.1
POST /search.php HTTP/1.1
curl -I http://example.local
