🧩 CCNA Packet Tracer: VLSM Design & Implementation

Write-up thiết kế địa chỉ IP bằng VLSM, chia subnet theo số host, cấu hình router/switch/PC và kiểm tra kết nối bằng ping/show command.

CCNAVLSMPacket TracerRouting

Mục tiêu

  • Biết chia subnet theo yêu cầu số host bằng VLSM.
  • Gán IP cho router interface, switch VLAN 1 và PC.
  • Kiểm tra kết nối end-to-end bằng ping, show ip interface brief và show running-config.

Nguyên tắc chia VLSM

  • Sắp xếp các mạng con theo số host giảm dần.
  • Chọn prefix nhỏ nhất nhưng đủ số host usable.
  • Cấp phát địa chỉ liên tục để tránh overlap.
  • Ghi rõ Network Address, Usable Range, Broadcast và Default Gateway.

Quy trình cấu hình

  • Cấu hình hostname cho router/switch.
  • Xóa IP sai nếu có bằng no ip address trước khi gán IP mới.
  • Gán IP cho các interface G0/0, G0/1, Serial theo bảng địa chỉ.
  • Bật interface bằng no shutdown.
  • Cấu hình default gateway cho switch và PC.
  • Lưu cấu hình bằng copy running-config startup-config.

Lệnh kiểm tra

  • show ip interface brief để kiểm tra trạng thái up/up và IP đã gán.
  • show running-config để xác nhận cấu hình.
  • ping default gateway từ PC.
  • ping giữa các PC khác subnet để kiểm tra routing.

Lỗi thường gặp

  • Nhầm subnet mask dẫn đến PC không cùng mạng với gateway.
  • Quên no shutdown trên router interface.
  • Gán nhầm gateway của PC.
  • Dùng subnet quá nhỏ so với số host yêu cầu.

Kỹ năng rút ra

  • Tư duy thiết kế địa chỉ IP logic.
  • Biết kiểm tra lỗi kết nối từ Layer 1 đến Layer 3.
  • Có nền tảng cho routing tĩnh, OSPF, ACL và NAT.

Command / Query tham khảo

show ip interface brief
show running-config
copy running-config startup-config
