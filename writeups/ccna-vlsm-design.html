1. Lab Objective

Mục tiêu của lab là ghi lại cách triển khai, kiểm thử và phân tích evidence theo hướng SOC Analyst. Bài viết không chỉ lưu command/cấu hình, mà phải trả lời: use case phát hiện gì, log đi qua đâu, evidence nào chứng minh alert và hạn chế hiện tại là gì.

2. Use Case & Log Flow

Endpoint / Traffic→Agent / Sensor→Manager / SIEM→Dashboard→Analyst Report
  • PC/Server kết nối switch access.
  • Switch chia VLAN và trunk về router/L3 switch.
  • Router xử lý routing, NAT hoặc ACL tùy lab.
  • Verification bằng ping, traceroute, show ip route, show vlan, show access-lists.

3. Configuration & Evidence Checklist

Evidence checklist: ghi lại cấu hình agent/rule, dashboard hoặc alert detail khi có kết quả lab. Tên file evidence nên thể hiện rõ nội dung như wazuh-rule-file-integrity.png hoặc kibana-portscan-dashboard.png.

4. Detection & Analysis

  • Không phải detection project; tập trung vào nền tảng để đọc network evidence.
  • ACL log có thể dùng như nguồn security event cơ bản.
  • Routing/VLAN sai có thể gây false symptom giống sự cố security.
FieldWhat to document
TimestampThời điểm alert xảy ra và múi giờ nếu có.
Source / DestinationIP, host, user, process hoặc email sender tùy case.
Rule / SignatureRule ID, description, severity và điều kiện match.
VerdictTrue positive, false positive, benign activity hoặc needs escalation.

5. Conclusion

Lab liên quan trực tiếp đến project CCNA Network Security Foundation. Kết luận cần ghi rõ rule phát hiện được gì, còn thiếu dữ liệu nào và action nào phù hợp nếu đây là môi trường thật.

  • Ghi lại command verification trong mỗi lab report.
  • Tạo bảng lỗi thường gặp: wrong gateway, wrong mask, shutdown interface, VLAN mismatch.
  • Liên hệ từng lab với use case SOC như firewall deny, scan, segmentation.
