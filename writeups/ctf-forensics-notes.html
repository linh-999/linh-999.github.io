1. Lab Objective

Mục tiêu của lab là ghi lại cách triển khai, kiểm thử và phân tích evidence theo hướng SOC Analyst. Bài viết không chỉ lưu command/cấu hình, mà phải trả lời: use case phát hiện gì, log đi qua đâu, evidence nào chứng minh alert và hạn chế hiện tại là gì.

2. Use Case & Log Flow

Endpoint / Traffic→Agent / Sensor→Manager / SIEM→Dashboard→Analyst Report
  • IDS sinh alert từ traffic capture.
  • PCAP được mở bằng Wireshark để follow stream.
  • Analyst filter theo IP/port/protocol/time.
  • Packet evidence được trích ra report.
  • Verdict được đưa ra kèm limitation.

3. Configuration / Evidence Placeholder

Chèn ảnh cấu hình agent/rule/dashboard/alert detail tại đây.

Khi thêm ảnh, nên đặt tên file rõ ràng như wazuh-rule-file-integrity.png hoặc kibana-portscan-dashboard.png.

4. Detection & Analysis

  • Port scan: nhiều SYN tới nhiều port/host.
  • Suspicious HTTP: user-agent lạ, path bất thường, payload encoded.
  • DNS anomaly: domain dài, query lặp, pattern giống tunneling.
  • Potential exfiltration: outbound data volume hoặc file transfer không bình thường.

VerdictTrue positive, false positive, benign activity hoặc needs escalation.

5. Conclusion

Lab liên quan trực tiếp đến project Network IDS & PCAP Investigation. Kết luận cần ghi rõ rule phát hiện được gì, còn thiếu dữ liệu nào và action nào phù hợp nếu đây là môi trường thật.

  • Lưu PCAP mẫu cho các case training.
  • Tạo cheat sheet filter Wireshark cho SOC L1.
  • Kết hợp IDS alert với DNS/proxy log nếu có.
