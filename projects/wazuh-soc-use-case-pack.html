01 · Executive Summary

Executive Summary

Xây dựng bộ use case Wazuh cho SOC L1 gồm File Integrity Monitoring, authentication failure, privileged command và suspicious process. Mỗi use case được mô tả bằng mục tiêu phát hiện, nguồn log, điều kiện sinh alert, bằng chứng mong đợi, khả năng false positive và hành động phản hồi.

Security operations value: Thể hiện khả năng chuyển yêu cầu giám sát thành rule/use case có thể vận hành và điều chỉnh noise.
02 · Scope

Objective & Scope

  • Theo dõi thay đổi file nhạy cảm trên endpoint.
  • Phát hiện đăng nhập thất bại bất thường và lệnh đặc quyền.
  • Chuẩn hóa mapping MITRE cho từng use case.
  • Thiết kế checklist triage cho SOC L1 khi nhận alert Wazuh.

05 · Triage process

Investigation Workflow

  1. Kiểm tra rule ID, agent name, hostname và severity.
  2. Đối chiếu raw log để tránh chỉ đọc mô tả alert.
  3. Kiểm tra user/host có nằm trong maintenance window hoặc admin activity không.
  4. Pivot sang các log trước/sau alert để xem chuỗi hành vi.
  5. Đưa verdict: benign admin activity, suspicious cần monitor hoặc escalate.

07 · Findings

Key Findings

  • Wazuh phù hợp cho lab SOC vì có agent, rule, dashboard và FIM tích hợp.
  • Use case cần được viết kèm điều kiện false positive để tránh alert overload.
  • Raw log quan trọng hơn phần mô tả alert vì mô tả có thể thiếu context.
08 · Response

Recommendations

  • Tách rule theo asset criticality và nhóm hệ thống.
  • Duy trì danh sách admin activity hợp lệ.
  • Tạo checklist triage cho từng nhóm alert.
  • Gắn MITRE mapping cho report để trình bày rõ hành vi tấn công.
09 · Continuous improvement

Limitations & Next Improvements

  • Tích hợp Sysmon cho Windows endpoint.
  • Tạo dashboard riêng cho authentication và privilege activity.
  • Viết correlation rule giữa failed login và successful login sau nhiều lần thất bại.
Quality note: Kết luận trong report luôn bám vào dữ liệu hiện có; nếu thiếu log hoặc thiếu context, phần limitation được ghi rõ để giữ tính chính xác của case.
