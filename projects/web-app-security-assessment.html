01 · summary

Executive Summary

Đây là project phù hợp mạnh với hướng Web/AppSec vì có đầy đủ vòng đời: phân tích chức năng, thiết kế database, triển khai ứng dụng, tích hợp dịch vụ ngoài và kiểm thử lỗ hổng. Nội dung được viết lại theo dạng security assessment thay vì chỉ là báo cáo môn học.

03 · scope

Assessment Scope

  • Broken Access Control: IDOR, Missing Authentication.
  • CSRF và session-sensitive action.
  • Security Misconfiguration: file upload, CORS/WebSocket origin, Clickjacking.
  • Injection: SQL Injection và Stored XSS.
  • Insecure Design và HTTPS/Transport Security hardening.

05 · findings

Finding Matrix

Finding groupRiskEvidence expectedRemediation direction
IDOR / Broken Access ControlHighUser A truy cập/sửa tài nguyên của User B qua object ID.Enforce object-level authorization ở service layer.
CSRFMediumSensitive action thiếu CSRF token hoặc origin validation.Enable CSRF protection, SameSite cookie, re-auth for critical actions.
File UploadHighUpload file không kiểm soát loại/extension/content.Validate MIME/content, randomize filename, store outside webroot.
CORS/WebSocket originMediumOrigin quá rộng hoặc WebSocket accept origin không kiểm soát.Restrict allowed origins and authenticate WebSocket handshake.
ClickjackingMediumThiếu X-Frame-Options/CSP frame-ancestors.Set security headers.
SQLi / Stored XSSHighPayload tác động query/rendered content.Parameterized query, output encoding, input validation.

07 · recommendations

Recommendations

  • Đưa authorization check vào service layer thay vì chỉ ẩn nút trên UI.
  • Chuẩn hóa security headers: CSP, HSTS, X-Frame-Options/frame-ancestors, X-Content-Type-Options.
  • Thiết kế upload pipeline an toàn và scan file nếu có thể.
  • Giới hạn CORS/WebSocket theo domain hợp lệ.
  • Duy trì checklist OWASP Top 10 cho mỗi release.
