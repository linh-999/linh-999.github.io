01 · Executive view

Executive Summary

Project tập trung vào cách xây dựng một hệ thống SIEM nhỏ nhưng có tư duy vận hành thật: endpoint/network sinh log, agent ship log, SIEM chuẩn hóa dữ liệu, dashboard hiển thị tín hiệu bất thường và analyst dùng evidence để kết luận alert. Điểm mạnh của project là không chỉ “cài tool”, mà trình bày được luồng log, rule phát hiện, cách phân loại alert, lý do false positive và hành động khuyến nghị.

Analyst value: Project này chứng minh được 4 năng lực quan trọng của SOC Analyst L1: hiểu log source, biết đọc alert field, biết pivot sang evidence, và không kết luận tấn công khi chưa kiểm tra context.

02 · Scope control

Objective & Scope

Objectives

  • Thiết kế luồng giám sát tập trung từ sensor/endpoint về SIEM dashboard.
  • Tạo các use case cơ bản: port scan, SSH brute force, suspicious outbound traffic và high-volume connection.
  • Viết mẫu report điều tra alert có timestamp, source, destination, evidence, verdict và recommendation.
  • Đánh giá hạn chế của rule khi thiếu threat intelligence hoặc endpoint context.

Scope boundary

  • Lab tập trung vào network-level detection, không khẳng định mức độ compromise nếu thiếu endpoint telemetry.
  • Các alert được đánh giá theo ngữ cảnh lab: traffic kiểm thử, host nội bộ, time range và rule match.
  • Không giả lập toàn bộ quy trình IR enterprise; chỉ mô phỏng triage và initial recommendation của SOC L1.
03 · Architecture

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
Endpoint / Attacker VM

Sinh traffic kiểm thử như scan port, SSH login failure, outbound connection.

Snort IDS Sensor

Giám sát traffic, match rule, sinh alert log gồm signature, protocol, source, destination.

Filebeat Agent

Thu thập alert log, thêm metadata và ship event về Elasticsearch.

Elasticsearch

Lưu trữ, index và hỗ trợ truy vấn theo timestamp, source IP, destination port, severity.

Kibana Dashboard

Hiển thị alert trend, top source IP, destination port, signature và event detail.

SOC Analyst

Pivot từ dashboard sang raw log, xác minh context và ghi verdict.

Ảnh cần chèn sau: topology lab, Kibana dashboard, Snort rule, alert detail hoặc command output. Không nên chèn ảnh minh họa mạng chung chung; ưu tiên ảnh có timestamp/log field thật từ lab.

06 · Triage workflow

Investigation Workflow

  1. Validate alert timestamp: kiểm tra alert xảy ra trong khoảng thời gian nào, có trùng với thời gian lab/test/maintenance hay không.
  2. Confirm spike on dashboard: xem dashboard có tăng bất thường theo time range, source IP hoặc destination port không.
  3. Pivot to raw event: lấy rule sid, signature, src.ip, dst.ip, dst.port, protocol, event count.
  4. Scope affected assets: xác định một host hay nhiều host bị ảnh hưởng, một source hay nhiều source, một service hay nhiều service.
  5. Check context: so sánh với traffic hợp lệ, danh sách scanner nội bộ, lab activity hoặc maintenance window.
  6. Classify verdict: true positive, benign positive, false positive hoặc needs escalation.
  7. Recommend action: monitor, block source, tune rule, enrich IOC hoặc escalate sang Tier 2.

08 · Analyst conclusion

Key Findings

Detection coverage

Hệ thống phát hiện được hành vi network-level rõ ràng như scan, brute force pattern và traffic spike.

Triage efficiency

Dashboard hỗ trợ analyst ưu tiên alert theo source, severity, destination port và time range.

False-positive awareness

Một số alert có thể là benign positive nếu đến từ scanner nội bộ hoặc hoạt động kiểm thử hợp lệ.

Context gap

Chưa đủ endpoint context để xác định process/user, nên không nên kết luận compromise chỉ dựa trên network alert.

09 · Response & tuning

Recommendations

  • Bổ sung asset inventory để phân biệt server quan trọng, máy lab, máy scanner và máy người dùng.
  • Tích hợp threat intelligence để enrich IP/domain trước khi kết luận malicious.
  • Thêm correlation với endpoint log để xác định process/user liên quan.
  • Xây allowlist cho scanner nội bộ và maintenance window để giảm noise.
  • Chuẩn hóa severity: không chỉ dựa vào rule priority, cần kết hợp asset criticality và scope ảnh hưởng.
  • Tạo dashboard riêng cho SOC L1 gồm: alert volume, top source, top destination, top signature, event detail và investigation checklist.
10 · Ready-to-submit report block

Final SOC Report Sample

Case title: Network-level suspicious activity detected by Snort IDS and visualized in ELK dashboard.

Severity: Medium in lab environment. Severity may increase if the destination is a critical server or if endpoint compromise evidence is confirmed.

Summary: During the lab time range, the SIEM dashboard recorded an abnormal increase in IDS alerts related to multiple connection attempts from the same source IP. The raw Snort alert included source IP, destination IP, destination port, protocol and rule SID, allowing the analyst to correlate the alert with port scanning/brute-force style behavior.

Verdict: Benign positive / lab-generated true positive. The detection logic worked as expected, but the alert should not be treated as confirmed compromise because endpoint telemetry and user/process context were not available.

Recommended action: Keep monitoring, document the source IP as lab scanner if authorized, tune detection thresholds, and enrich future alerts with asset inventory and endpoint telemetry.

11 · Continuous improvement

Limitations & Next Improvements

  • Thêm Suricata/Zeek để tăng context network, đặc biệt là DNS, HTTP, TLS và connection metadata.
  • Viết detection rule mapping MITRE rõ hơn và thêm risk score cho từng use case.
  • Tự động sinh ticket summary từ alert fields: time, source, destination, evidence, verdict, recommendation.
  • Tích hợp Sysmon hoặc EDR để xác định process/user tạo traffic.
  • Thêm baseline theo khung giờ để giảm false positive khi có traffic hợp lệ tăng cao.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
