Centralized IDS Monitoring with Snort & ELK

Triển khai hệ thống IDS mã nguồn mở dựa trên Snort, tích hợp log về ELK Stack qua Filebeat và trình bày alert trên Kibana. Project được viết lại theo hướng SOC/SIEM: từ kiến trúc thu thập log, logic detection, evidence, false positive đến đề xuất tuning.

01 · Executive summary

Executive Summary

Project mô phỏng một hệ thống giám sát bảo mật tập trung cho mạng lab. Snort đóng vai trò network IDS sensor, Filebeat ship log, Elasticsearch lưu trữ/chỉ mục và Kibana hỗ trợ dashboard/triage.

Security value: Chứng minh khả năng hiểu luồng log end-to-end, không chỉ cài tool mà biết giải thích vì sao alert sinh ra và analyst cần kiểm tra gì tiếp theo.
02 · scope

Objective & Scope

  • Xây dựng pipeline phát hiện xâm nhập dựa trên Snort và ELK.
  • Thiết kế/áp dụng rule cho các kịch bản tấn công phổ biến trong lab.
  • Tạo dashboard hỗ trợ phân tích theo timestamp, source, destination, rule và severity.
  • Ghi rõ limitation khi thiếu endpoint telemetry hoặc asset inventory.
03 · architecture

05 · workflow

Investigation Workflow

  1. Xác định alert time window và rule SID.
  2. Pivot sang raw event để lấy source IP, destination IP, port, protocol và payload nếu có.
  3. Xác định phạm vi: một host hay nhiều host, một rule hay nhiều rule liên quan.
  4. Đối chiếu với kịch bản lab/maintenance để phân loại true positive, benign test hoặc false positive.
  5. Viết verdict kèm confidence, limitation và recommended action.

07 · findings

Findings & Recommendations

Finding: Pipeline phù hợp để học SIEM use case cơ bản, đặc biệt là cách đi từ network alert đến report điều tra có cấu trúc.
  • Bổ sung Zeek/Suricata để tăng DNS/HTTP/TLS context.
  • Bổ sung endpoint log để xác định process/user tạo traffic.
  • Mapping rule sang MITRE ATT&CK để report chuyên nghiệp hơn.
  • Tách dashboard cho scan, web attack, DoS và outbound anomaly.
08 · improvements

Limitations & Next Improvements

  • Chưa có asset criticality nên severity còn phụ thuộc nhiều vào rule.
  • Chưa có threat intelligence enrichment cho IP/domain bên ngoài.
  • Chưa có ticket template tự động từ alert fields.
  • Cần bổ sung rule tuning theo allowlist scanner nội bộ.
