Executive Summary

Project tập trung vào cách xây dựng một hệ thống SIEM nhỏ nhưng có tư duy vận hành thật: endpoint/network sinh log, agent ship log, SIEM chuẩn hóa dữ liệu, dashboard hiển thị tín hiệu bất thường và analyst dùng evidence để kết luận alert. Điểm mạnh của project là không chỉ “cài tool”, mà trình bày được luồng log, rule phát hiện, cách phân loại alert, lý do false positive và hành động khuyến nghị.

Objective & Scope

  • Thiết kế luồng giám sát tập trung từ sensor/endpoint về SIEM dashboard.
  • Tạo các use case cơ bản: port scan, brute force, suspicious outbound traffic, high-volume connection.
  • Viết mẫu report điều tra alert có timestamp, source, destination, evidence, verdict và recommendation.
  • Đánh giá hạn chế của rule khi thiếu threat intelligence hoặc endpoint context.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • Endpoint / attacker VM tạo traffic kiểm thử.
  • Snort IDS giám sát traffic và sinh alert theo rule.
  • Filebeat thu thập alert log và đẩy về Elasticsearch.
  • Kibana dashboard trực quan hóa alert theo severity, source IP, destination port và time range.
  • SOC analyst thực hiện triage dựa trên dashboard, raw event và packet/log evidence.
Chèn ảnh lab tại đây: topology, dashboard, rule, event detail hoặc command output.

Detection Logic / Use Cases

  • Port scanning: nhiều connection tới nhiều destination port trong thời gian ngắn.
  • SSH brute force: nhiều authentication failure từ một source IP.
  • Suspicious outbound: kết nối tới IP/domain lạ hoặc port không phổ biến.
  • Potential DoS: spike traffic bất thường so với baseline lab.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Validate alert timestamp và kiểm tra dashboard có spike bất thường hay không.
  2. Pivot sang raw event để lấy rule sid, source IP, destination IP, port, protocol.
  3. Kiểm tra scope: một host hay nhiều host bị ảnh hưởng, một user hay nhiều user liên quan.
  4. So sánh với traffic lab hợp lệ để phân loại true positive, benign test hoặc false positive.
  5. Ghi verdict và đề xuất action: monitor, block source, tune rule hoặc escalate.

Key Findings

  • Hệ thống phát hiện được hành vi network-level rõ ràng như scan và brute force.
  • Dashboard hỗ trợ analyst ưu tiên alert theo source, severity và time range.
  • Một số alert có thể false positive nếu traffic đến từ scanner nội bộ hoặc hoạt động kiểm thử hợp lệ.

Recommendations

  • Bổ sung asset inventory để phân biệt server quan trọng và máy lab.
  • Tích hợp threat intelligence để enrich IP/domain trước khi kết luận.
  • Thêm correlation với endpoint log để xác định process/user liên quan.
  • Xây allowlist cho scanner nội bộ và maintenance window để giảm noise.

Limitations & Next Improvements

  • Thêm Suricata/Zeek để tăng context network.
  • Viết detection rule mapping MITRE rõ hơn.
  • Tự động sinh ticket summary từ alert fields.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
