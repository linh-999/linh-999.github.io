📊 Centralized Security Monitoring System

Thiết kế mô hình giám sát bảo mật tập trung theo tư duy SOC Analyst: thu thập log, phát hiện bất thường, triage alert và viết báo cáo điều tra.

Snort IDSELK StackFilebeatLinuxSOC Triage
RoleSOC Analyst / Detection Lab Builder
GoalPhát hiện port scan, DoS và network anomaly trong môi trường lab.
OutcomeCó dashboard, alert flow, rule logic và checklist điều tra.

1. Problem Statement

Trong môi trường doanh nghiệp, log rời rạc từ nhiều endpoint và thiết bị mạng khiến analyst khó phát hiện hành vi bất thường. Project này mô phỏng cách gom log về một điểm trung tâm để SOC có thể giám sát, truy vấn và điều tra nhanh.

2. Architecture & Log Flow

Endpoint / Attacker VMSinh traffic scan, DoS, connection attempt
Snort IDSMatch rule, tạo alert
FilebeatForward IDS log
ELK StackIndex, parse, search
Kibana DashboardMonitoring & investigation

3. Detection Use Cases

Use caseDetection logicAnalyst value
Port ScanningNhiều connection attempts đến nhiều port trong thời gian ngắn.Nhận diện reconnaissance trước khi khai thác.
DoS AttemptLưu lượng bất thường, nhiều packet đến cùng một destination.Đánh giá nguy cơ ảnh hưởng availability.
Suspicious External IPIP lạ liên tục tương tác với host nội bộ.Pivot sang threat intelligence và firewall log.

4. SOC Analyst Workflow

Triage

Kiểm tra rule name, severity, source IP, destination IP, timestamp và tần suất alert.

Context Collection

Pivot sang dashboard để xem traffic trước/sau alert, các host liên quan và pattern lặp lại.

Assessment

Phân loại true positive / false positive dựa trên bằng chứng, whitelist, lab activity hoặc traffic hợp lệ.

Recommendation

Đề xuất block IP, tune rule, tăng logging hoặc tạo dashboard theo dõi riêng.

5. Evidence cần đưa vào portfolio

Ảnh nên chèn

Topology lab, Snort rule, log alert, dashboard Kibana, query lọc theo source IP/rule, kết quả ping/nmap dùng để tái hiện alert.

6. Kết quả & Impact

VisibilityTập trung log để giảm phụ thuộc kiểm tra thủ công.
DetectionCó rule phát hiện scan/DoS trong lab.
TriageCó checklist xử lý alert bước đầu.
ReportingViết được kết luận có evidence.

7. Limitations & Improvement

  • Rule lab còn đơn giản, chưa bao phủ đầy đủ kỹ thuật tấn công thực tế.
  • Cần thêm enrichment với threat intelligence để đánh giá IP/domain tốt hơn.
  • Cần mapping use case với MITRE ATT&CK để chuyên nghiệp hóa detection report.
