01 · Executive Summary

Executive Summary

Thiết kế một pipeline giám sát bảo mật tập trung để chuyển cảnh báo network-level thành case triage có cấu trúc. Project không dừng ở việc cài Snort/ELK, mà tập trung vào cách analyst đọc alert, kiểm tra raw event, xác định source/destination, so sánh với baseline lab và đưa verdict có bằng chứng.

Security operations value: Chứng minh khả năng hiểu luồng log end-to-end và trình bày một SIEM use case theo ngôn ngữ vận hành SOC.
02 · Scope

Objective & Scope

  • Thiết kế luồng thu thập alert từ sensor về dashboard tập trung.
  • Xây dựng use case phát hiện port scanning, SSH brute force, suspicious outbound traffic và high-volume connection.
  • Chuẩn hóa format report gồm timestamp, source, destination, evidence, verdict và recommended action.
  • Ghi rõ điểm giới hạn khi thiếu endpoint telemetry hoặc threat intelligence.

05 · Triage process

Investigation Workflow

  1. Validate alert timestamp và xác định time window cần điều tra.
  2. Pivot từ dashboard sang raw event để lấy rule SID, source IP, destination IP, port và protocol.
  3. Kiểm tra scope: một host hay nhiều host, một destination hay nhiều destination.
  4. Đối chiếu với traffic lab hợp lệ để phân loại true positive, benign test hoặc false positive.
  5. Ghi verdict kèm confidence, limitation và action: monitor, tune rule, block source hoặc escalate.

07 · Findings

Key Findings

  • Hệ thống phát hiện tốt các hành vi network-level rõ ràng như scan, brute force và traffic spike.
  • Dashboard hỗ trợ ưu tiên alert theo source IP, severity và time range.
  • False positive có thể xuất hiện nếu scanner nội bộ hoặc maintenance traffic chưa được allowlist.
  • Project thể hiện được vòng đời SIEM use case: data source → rule → alert → triage → tuning.
08 · Response

Recommendations

  • Bổ sung asset inventory để phân biệt server quan trọng, máy người dùng và máy lab.
  • Tích hợp threat intelligence để enrich external IP/domain trước khi kết luận.
  • Correlation thêm endpoint log để xác định process/user tạo traffic.
  • Duy trì allowlist có owner và thời hạn cho scanner nội bộ hoặc maintenance window.
09 · Continuous improvement

Limitations & Next Improvements

  • Thêm Zeek/Suricata để tăng network context như DNS, HTTP, TLS và JA3.
  • Mapping mỗi use case với MITRE ATT&CK tactic/technique.
  • Tạo template ticket tự động từ các trường alert quan trọng.
  • Định nghĩa severity dựa trên asset criticality và confidence thay vì chỉ dựa vào rule name.
Quality note: Kết luận trong report luôn bám vào dữ liệu hiện có; nếu thiếu log hoặc thiếu context, phần limitation được ghi rõ để giữ tính chính xác của case.
