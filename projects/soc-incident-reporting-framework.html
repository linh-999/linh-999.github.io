Executive Summary

Project này đóng vai trò “xương sống” cho toàn bộ portfolio: một framework giúp mọi lab/report nhìn thống nhất và chuyên nghiệp. Điểm quan trọng là mỗi kết luận đều đi kèm bằng chứng, confidence và limitation, tránh viết chung chung.

Objective & Scope

  • Chuẩn hóa cấu trúc report SOC cho project và write-up.
  • Tạo bảng evidence để người đọc kiểm chứng kết luận.
  • Tách rõ finding, impact, recommendation và limitation.
  • Dùng cùng format cho phishing, endpoint, network, SIEM và WAF case.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • Alert intake nhận thông tin ban đầu.
  • Evidence collection thu thập log/screenshot/artifact.
  • Timeline reconstruction sắp xếp event theo thời gian.
  • Analysis & verdict đánh giá true positive/false positive.
  • Response recommendation đưa hành động tiếp theo.
Evidence artifacts: topology, dashboard, rule detail, raw event và command output được quản lý như bằng chứng kỹ thuật của case.

Detection Logic / Use Cases

  • Framework không phát hiện trực tiếp, nhưng làm rõ chất lượng detection.
  • Mỗi detection được mô tả bằng objective, data source, logic và expected evidence.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Viết executive summary 3–5 dòng.
  2. Ghi scope: user, host, IP, asset, time range.
  3. Liệt kê evidence có nguồn rõ ràng.
  4. Lập timeline nếu có nhiều event.
  5. Đưa verdict kèm confidence và limitation.
  6. Đề xuất response action cụ thể.

Key Findings

  • Report tốt giúp analyst thể hiện tư duy hơn là chỉ liệt kê tool.
  • Evidence table làm portfolio đáng tin và dễ review.
  • Limitation thể hiện sự chuyên nghiệp vì không phỏng đoán quá mức.

Recommendations

  • Dùng cùng template cho mọi project.
  • Thêm screenshot thật khi có lab.
  • Tách rõ “observed evidence” và “analyst assessment”.

Limitations & Next Improvements

  • Tạo markdown template reusable.
  • Thêm severity matrix.
  • Thêm executive one-page report cho hiring manager.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
