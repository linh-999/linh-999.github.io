Executive Summary

Project này chứng minh khả năng đọc Windows authentication log và biến event rời rạc thành một detection logic có ý nghĩa. Analyst không chỉ thấy 4625 là “login fail”, mà phải hiểu pattern theo user, source IP, thời gian và kết quả sau đó có 4624 thành công hay account lockout không.

Objective & Scope

  • Thiết kế logic phát hiện brute force và password spraying.
  • Phân biệt sai mật khẩu đơn lẻ, user quên mật khẩu và attack pattern.
  • Tạo triage checklist cho authentication alert.
  • Đưa ra khuyến nghị response phù hợp với mức độ rủi ro.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • Windows host hoặc domain controller sinh Security Event Log.
  • Event được forward về SIEM/Wazuh/ELK.
  • Rule correlation tính số lần fail theo source/user/time window.
  • Analyst kiểm tra event sequence và context.
  • Verdict được ghi kèm action: monitor, block, reset password hoặc escalate.
Chèn ảnh lab tại đây: topology, dashboard, rule, event detail hoặc command output.

Detection Logic / Use Cases

  • Multiple 4625 từ một source IP tới nhiều username trong time window ngắn.
  • Nhiều 4625 vào một username từ nhiều source IP khác nhau.
  • 4625 liên tiếp sau đó có 4624 success từ cùng source, gợi ý credential compromise.
  • 4740 account lockout xảy ra sau chuỗi failed login.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Xác định event ID, logon type, username, source workstation/IP.
  2. Đếm số failed login theo user/source trong 5–15 phút.
  3. Kiểm tra có 4624 success sau chuỗi fail không.
  4. Kiểm tra tài khoản có thuộc nhóm privileged hoặc critical không.
  5. Phân loại: benign user error, brute force, password spraying hoặc compromised account.

Evidence Collected

EvidenceAnalyst value

Event ID 4624Successful login sau nhiều lần fail là tín hiệu cần ưu tiên.
Event ID 4740Account lockout cho thấy tác động tới user/account.
PatternNhiều user từ một source thường nghi password spraying hơn user error.

Key Findings

  • Event 4625 đơn lẻ chưa đủ để kết luận tấn công.
  • Correlation theo time window giúp giảm false positive.
  • Privileged account cần severity cao hơn account thường.

Recommendations

  • Áp dụng threshold theo baseline từng môi trường.
  • Alert riêng cho privileged account.
  • Kết hợp VPN/firewall log để xác định nguồn ngoài hay nội bộ.
  • Bật MFA và account lockout policy hợp lý.

Limitations & Next Improvements

  • Viết Sigma-style detection logic.
  • Tạo dashboard failed/success login theo user/source.
  • Thêm enrichment từ danh sách privileged users.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
