01 · Executive Summary

Executive Summary

Bộ lab Packet Tracer dùng để rèn nền tảng mạng: VLSM, VLAN, routing, switch management, ACL và troubleshooting. Nội dung được viết lại theo hướng portfolio: mỗi lab có mục tiêu, topology, addressing plan, cấu hình chính, verification command và lỗi thường gặp.

Security operations value: Giúp nhà tuyển dụng thấy nền tảng network đủ để đọc alert và làm việc với firewall/router/switch trong môi trường SOC/NOC.
02 · Scope

Objective & Scope

  • Thực hành chia subnet theo VLSM chính xác.
  • Cấu hình VLAN, gateway và routing cơ bản.
  • Kiểm tra kết nối end-to-end bằng lệnh phù hợp.
  • Ghi lỗi cấu hình thường gặp và cách sửa.
03 · Data flow

Luồng trên giúp case có điểm bắt đầu và điểm kết thúc rõ ràng: nguồn dữ liệu sinh event, hệ thống xử lý/correlation và analyst đưa kết luận dựa trên bằng chứng.

04 · Detection design

Detection / Analysis Logic

Use caseConditionExpected evidenceMITRE / Analyst mappingFalse-positive consideration
AddressingIP/mask/gateway đúng network.Address table, ping test.FoundationSai mask hoặc gateway.
SwitchingVLAN và access port đúng.show vlan brief, show interfaces status.SegmentationCắm sai port hoặc VLAN thiếu.
RoutingRoute đến remote network tồn tại.show ip route, traceroute.ConnectivityDefault route thiếu.
ACLRule permit/deny đúng yêu cầu.show access-lists, hit count.Policy ControlSai thứ tự rule.
05 · Triage process

Investigation Workflow

  1. Đọc yêu cầu host/subnet.
  2. Thiết kế addressing table trước khi cấu hình.
  3. Cấu hình thiết bị theo từng lớp: host, switch, router, ACL.
  4. Verify từng bước bằng command evidence.
  5. Ghi troubleshooting notes nếu ping/traceroute thất bại.

07 · Findings

Key Findings

  • Thiết kế IP trước giúp giảm lỗi cấu hình.
  • Troubleshooting cần đi theo lớp: physical → VLAN → IP → route → ACL.
  • Network foundation hỗ trợ rất nhiều khi điều tra alert có source/destination phức tạp.
08 · Response

Recommendations

  • Mỗi lab nên lưu addressing table và verification output.
  • Ghi rõ lỗi gặp phải và nguyên nhân để thể hiện tư duy troubleshooting.
  • Liên hệ lab network với use case security như segmentation và ACL.
09 · Continuous improvement

Limitations & Next Improvements

  • Thêm lab OSPF, DHCP, NAT và inter-VLAN routing.
  • Mô phỏng firewall rule và network segmentation.
  • Tạo sơ đồ topology rõ ràng cho từng case.
