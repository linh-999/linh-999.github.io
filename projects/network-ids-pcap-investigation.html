Executive Summary

Project này thể hiện nền tảng network security quan trọng cho SOC: đọc packet, hiểu TCP/IP, xác định session, decode payload cơ bản và dùng PCAP để validate alert IDS. Đây là kỹ năng giúp analyst không bị phụ thuộc hoàn toàn vào dashboard.

Objective & Scope

  • Phân tích traffic từ alert IDS đến packet-level evidence.
  • Xác định source, destination, port, protocol, session và payload liên quan.
  • Phân biệt scan, normal browsing, suspicious beaconing hoặc file transfer.
  • Viết kết luận dựa trên packet evidence.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • IDS sinh alert từ traffic capture.
  • PCAP được mở bằng Wireshark để follow stream.
  • Analyst filter theo IP/port/protocol/time.
  • Packet evidence được trích ra report.
  • Verdict được đưa ra kèm limitation.
Evidence artifacts: topology, dashboard, rule detail, raw event và command output được quản lý như bằng chứng kỹ thuật của case.

Detection Logic / Use Cases

  • Port scan: nhiều SYN tới nhiều port/host.
  • Suspicious HTTP: user-agent lạ, path bất thường, payload encoded.
  • DNS anomaly: domain dài, query lặp, pattern giống tunneling.
  • Potential exfiltration: outbound data volume hoặc file transfer không bình thường.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Bắt đầu từ alert metadata: timestamp, sid, src/dst IP, port.
  2. Filter PCAP theo tuple và time range.
  3. Follow TCP stream để hiểu request/response.
  4. Kiểm tra handshake, response code, payload và bytes transferred.
  5. Kết luận alert có evidence đủ mạnh hay cần thêm endpoint/proxy log.

Key Findings

  • PCAP giúp validate alert và giảm false positive.
  • Cần hiểu protocol để không đọc sai packet.
  • Network evidence thường cần bổ sung endpoint context để xác định user/process.

Recommendations

  • Lưu PCAP mẫu cho các case training.
  • Tạo cheat sheet filter Wireshark cho SOC L1.
  • Kết hợp IDS alert với DNS/proxy log nếu có.

Limitations & Next Improvements

  • Tự động extract IOC từ PCAP.
  • Thêm Zeek log để dễ pivot hơn.
  • Viết report template cho từng traffic pattern.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
