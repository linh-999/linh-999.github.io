01 · Executive Summary

Executive Summary

Phân tích IDS alert và PCAP để xác định bản chất traffic: reconnaissance, exploit attempt, malware download hay benign scan. Project tập trung vào kỹ năng đọc packet, follow TCP stream, kiểm tra DNS/HTTP/TLS và liên kết packet evidence với alert.

Security operations value: Tăng năng lực điều tra network-level alert khi SIEM chỉ cung cấp một phần context.
02 · Scope

Objective & Scope

  • Xác minh IDS alert bằng packet evidence.
  • Extract IOC từ DNS, HTTP Host, URL, SNI hoặc IP.
  • Kiểm tra payload/response để đánh giá impact.
  • Viết report có timestamp, flow và conclusion rõ ràng.

05 · Triage process

Investigation Workflow

  1. Xác định alert time và 5-tuple network flow.
  2. Lọc PCAP theo source/destination/protocol/time window.
  3. Follow stream để đọc request/response nếu giao thức cho phép.
  4. Extract IOC và đối chiếu với DNS/HTTP/TLS context.
  5. Kết luận behavior và ghi rõ limitation nếu payload bị mã hóa.

07 · Findings

Key Findings

  • PCAP giúp validate alert chính xác hơn dashboard summary.
  • Traffic mã hóa vẫn có thể phân tích metadata như SNI, timing và byte pattern.
  • Một flow đáng nghi cần được đặt trong bối cảnh host và user để giảm false positive.
08 · Response

Recommendations

  • Lưu PCAP quanh thời điểm alert cho các host critical.
  • Dùng Zeek để tự động hóa DNS/HTTP/TLS metadata.
  • Chuẩn hóa bảng IOC extraction cho mọi network investigation.
09 · Continuous improvement

Limitations & Next Improvements

  • Tạo filter library cho Wireshark theo use case.
  • Kết hợp IDS alert với Zeek logs và endpoint telemetry.
  • Xây dựng baseline cho internal scanner và service traffic.
Quality note: Kết luận trong report luôn bám vào dữ liệu hiện có; nếu thiếu log hoặc thiếu context, phần limitation được ghi rõ để giữ tính chính xác của case.
