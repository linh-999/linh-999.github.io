🌐 Web Security & WAF Mitigation Lab

Phân tích web attack dưới góc nhìn SOC Analyst: payload, WAF event, web access log, false positive và khuyến nghị mitigation.

Executive Summary

Project mô phỏng quy trình SOC/Web Security khi nhận các event từ Web Application Firewall. Mục tiêu là không chỉ kiểm thử payload OWASP, mà còn đọc log WAF, hiểu rule nào kích hoạt, kiểm tra request gốc, đánh giá có phải tấn công thật hay false positive và viết khuyến nghị phòng thủ.

1. Scope

Phạm vi lab gồm các nhóm lỗi phổ biến: SQL Injection, reflected XSS, command injection pattern, file upload risk và brute-force login. Các payload được tạo trong môi trường lab để quan sát WAF event, không áp dụng trên hệ thống thật khi chưa được phép.

4. Evidence Collection

  • Request raw trong Burp Suite: method, endpoint, parameter, payload.
  • WAF event: rule ID, action block/monitor, severity, source IP, URI.
  • Web access log: status code, response size, user-agent, timestamp.
  • Application behavior: payload có thực thi hay chỉ bị ghi log.

6. SOC Analyst Conclusion

Lab chứng minh WAF có thể phát hiện nhiều payload phổ biến, nhưng analyst không nên kết luận “bị tấn công thành công” chỉ dựa trên alert. Cần đối chiếu WAF action, web status code, server log và application behavior. Một alert bị block vẫn là một security event, nhưng impact thấp hơn so với payload bypass và tới được backend.

7. Limitations & Improvement

  • WAF signature-based detection dễ bị bypass nếu payload encode hoặc biến thể phức tạp.
  • Cần log correlation với authentication log để đánh giá account compromise.
  • Nên thêm dashboard theo top URI, top rule ID, top source IP, action block/allow.
  • Nâng cấp project bằng cách viết playbook: WAF SQLi Alert, WAF XSS Alert, Brute Force Web Login.
