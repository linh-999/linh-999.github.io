Executive Summary

Project thể hiện khả năng kết nối giữa pentest cơ bản và SOC monitoring. Analyst cần hiểu payload web attack để đọc WAF log đúng: request nào thật sự nguy hiểm, request nào là scanner/lab traffic, rule nào block đúng và rule nào cần tuning.

Objective & Scope

  • Thực hành nhận diện SQLi, XSS, file upload và command injection pattern.
  • Đọc WAF log để xác định rule ID, action, payload và source.
  • Viết report mitigation dựa trên evidence thay vì chỉ nói “bị tấn công”.
  • Đề xuất hardening phía application và tuning phía WAF.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • Attacker/lab browser gửi request qua Burp Suite.
  • WAF kiểm tra request và ghi event block/alert.
  • Web server ghi access log và response code.
  • SOC analyst correlate WAF event với access log để xác nhận impact.
  • Report đưa ra finding và mitigation.
Chèn ảnh lab tại đây: topology, dashboard, rule, event detail hoặc command output.

Detection Logic / Use Cases

  • SQL injection pattern: quote, union select, boolean-based payload.
  • XSS pattern: script tag, event handler, encoded payload.
  • File upload risk: extension bypass, content-type mismatch.
  • Command injection: shell metacharacters và command keyword bất thường.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Xác định URL, method, source IP, user-agent và payload.
  2. Kiểm tra WAF action: blocked, alerted, allowed hay bypass.
  3. Đối chiếu web access log để xem request có tới backend không.
  4. Đánh giá response code và dấu hiệu data exposure/execution.
  5. Phân loại severity và đề xuất mitigation.

Key Findings

  • WAF giúp giảm rủi ro nhưng không thay thế secure coding.
  • Một số payload encoded có thể cần rule tuning.
  • Correlation WAF + access log giúp tránh kết luận sai.

Recommendations

  • Validate input phía application và dùng parameterized query.
  • Bật logging đầy đủ cho URL, source IP, user-agent và rule ID.
  • Tạo allowlist cẩn thận để tránh bypass.
  • Review false positive theo endpoint nghiệp vụ.

Limitations & Next Improvements

  • Thêm ModSecurity CRS lab.
  • Viết checklist đọc WAF alert cho SOC L1.
  • Tạo dashboard top attack path/payload.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
