01 · Executive Summary

Executive Summary

Lab kiểm thử các pattern OWASP Top 10 trên ứng dụng web và phân tích event từ WAF/access log. Project tập trung vào cách SOC/Web Security Analyst đọc request, xác định rule match, kiểm tra response, phân loại blocked/allowed traffic và đề xuất tuning rule.

Security operations value: Thể hiện khả năng điều tra alert web ở cả góc nhìn offensive testing và defensive monitoring.
02 · Scope

Objective & Scope

  • Kiểm tra các pattern SQL injection, XSS, path traversal và suspicious user-agent.
  • Đối chiếu WAF event với web access log để xác định impact.
  • Phân biệt true attack, blocked attempt và false positive.
  • Viết khuyến nghị mitigation dựa trên evidence.

05 · Triage process

Investigation Workflow

  1. Xác định WAF rule ID, action và target endpoint.
  2. Kiểm tra raw request để hiểu payload và parameter bị rule match.
  3. Đối chiếu web access log để biết request bị block hay backend trả response.
  4. Kiểm tra số lần lặp lại, source IP và các endpoint liên quan.
  5. Đề xuất block/tune rule hoặc xác nhận false positive với app owner.

07 · Findings

Key Findings

  • WAF block không đồng nghĩa ứng dụng đã bị compromise; cần xem response và backend log.
  • Một số payload có thể là kiểm thử hợp lệ nếu đến từ lab/scanner nội bộ.
  • Rule tuning cần dựa trên endpoint và parameter cụ thể để tránh mở rộng rủi ro.
08 · Response

Recommendations

  • Duy trì whitelist scanner nội bộ theo IP và lịch kiểm thử.
  • Tạo dashboard WAF theo rule ID, endpoint và source IP.
  • Phối hợp app owner khi cần tune false positive trên parameter hợp lệ.
09 · Continuous improvement

Limitations & Next Improvements

  • Thêm correlation giữa WAF, app log và authentication log.
  • Phân loại endpoint criticality để ưu tiên alert.
  • Tạo playbook riêng cho SQLi, XSS và path traversal.
Quality note: Kết luận trong report luôn bám vào dữ liệu hiện có; nếu thiếu log hoặc thiếu context, phần limitation được ghi rõ để giữ tính chính xác của case.
