FortiWeb WAF Defense Lab

Triển khai FortiWeb-100D trong vai trò Web Application Firewall, cấu hình đối tượng máy chủ và kiểm thử các nhóm chính sách bảo vệ ứng dụng web. Nội dung được trình bày theo góc nhìn Web Security Analyst: request nào bị rule match, WAF xử lý ra sao, evidence là gì và cần tune gì để giảm false positive.

01 · summary

Executive Summary

Lab tập trung vào cách WAF bảo vệ ứng dụng trước request độc hại và cách analyst đọc event để biết request bị chặn vì lý do gì. Điểm quan trọng không phải chỉ là “WAF block”, mà là hiểu rule, payload, endpoint và impact.

02 · scope

Objective & Scope

  • Cấu hình server object và policy bảo vệ web application.
  • Kiểm thử Known Attacks: XSS, SQL Injection, XXE.
  • Kiểm thử Cookie Security và URL Access Control.
  • Phân tích vấn đề gặp phải, nguyên nhân thất bại/thành công và hướng tuning.
03 · architecture

Architecture / Traffic Flow

Analyst cần đọc cả WAF event và backend/access log để xác định request bị chặn trước WAF hay đã đi vào ứng dụng.

05 · workflow

Investigation Workflow

  1. Xác định WAF event: timestamp, source IP, endpoint, rule ID và action.
  2. Mở raw request để xem payload/parameter bị match.
  3. Đối chiếu access log để biết request có tới backend không.
  4. Kiểm tra tần suất và các endpoint liên quan để đánh giá campaign hay event đơn lẻ.
  5. Kết luận blocked attack, allowed suspicious request, false positive hoặc need tuning.

07 · findings

Findings

  • WAF hữu ích để chặn nhiều pattern web attack phổ biến nhưng cần tuning theo endpoint.
  • Block event chưa đủ để kết luận compromise; phải kiểm tra backend response/log.
  • Các rule nhạy cảm như SQLi/XSS cần quản lý exception theo parameter, không mở rộng toàn site.
08 · recommendations

Recommendations

  • Tạo dashboard theo rule ID, endpoint, source IP và action.
  • Duy trì whitelist scanner nội bộ theo thời gian kiểm thử.
  • Kết hợp WAF event với authentication log và application error log.
  • Định nghĩa severity theo endpoint criticality và action thực tế.
09 · improvements

Limitations & Improvements

  • Cần thêm mô hình DevSecOps để fix lỗi ở code, không chỉ chặn bằng WAF.
  • Cần test ở chế độ alert-only trước khi block production.
  • Cần quy trình review exception định kỳ để tránh mở lỗ hổng lâu dài.
