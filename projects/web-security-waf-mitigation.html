🧱 Web Security: Penetration Testing & WAF Mitigation

Phân tích web attack theo góc nhìn SOC: từ payload kiểm thử, WAF alert, web log evidence đến đánh giá rủi ro và khuyến nghị giảm thiểu.

FortiWeb WAFBurp SuiteOWASP ZAPDVWAWeb Logs
RoleSecurity Analyst focusing on Web/WAF log review
GoalHiểu payload, rule match và bằng chứng trong HTTP log.
OutcomeBiết trình bày vulnerability + WAF mitigation theo format báo cáo.

1. Objective

Dự án mô phỏng quy trình kiểm thử một ứng dụng web lab, sau đó phân tích log từ WAF/web server để xác định request nào bị chặn, request nào chỉ được ghi nhận và rule nào cần tuning.

2. Analysis Flow

Browser / BurpGửi request kiểm thử
DVWA / Web AppEndpoint nhận payload
FortiWeb WAFDetect / Block / Log
Web LogsStatus code, URI, parameter
Analyst ReportEvidence, impact, recommendation

3. Use Cases

Attack classEvidence cần kiểm traRecommendation
SQL InjectionURI, parameter, payload pattern, DB error, WAF signature.Parameterized query, input validation, WAF signature tuning.
Broken Access ControlUser role, object ID, response code, unauthorized access attempt.Server-side authorization, deny-by-default policy, audit log.
XSSScript payload, reflected/stored location, response body context.Output encoding, CSP, input sanitization.

4. SOC-style Investigation Notes

  • Xác định alert là true positive khi payload khớp rule và request có dấu hiệu khai thác rõ ràng.
  • Xác định false positive khi request hợp lệ nhưng bị rule nhận nhầm do pattern quá rộng.
  • Luôn ghi rõ source IP, user-agent, URI, parameter, HTTP method, status code và thời điểm xảy ra.
  • Đối chiếu WAF log với web access/error log để biết request có thật sự đi tới ứng dụng hay đã bị chặn trước.

5. Report Output

Description

Mô tả ngắn lỗ hổng hoặc hành vi tấn công được quan sát.

Evidence

Ảnh Burp request, WAF event, web log, status code và payload đã mask nếu cần.

Impact

Đánh giá ảnh hưởng đến confidentiality, integrity hoặc availability.

Mitigation

Đề xuất fix code, hardening config, WAF rule và monitoring dashboard.

6. Limitations & Improvement

  • Lab chưa đại diện toàn bộ môi trường production, cần thêm authentication flow và real application logs.
  • Cần chuẩn hóa severity theo CVSS hoặc risk matrix nội bộ.
  • Cần mapping OWASP Top 10 với detection rule để portfolio mạnh hơn.
