🤖 AI-Driven Cybersecurity Agent for SOC/NOC Automation

Thiết kế workflow hỗ trợ analyst: parse alert, enrich IOC, sinh checklist điều tra và tạo bản nháp báo cáo có kiểm soát.

Executive Summary

Project thiết kế một assistant hỗ trợ SOC/NOC Analyst trong các bước lặp lại: đọc alert thô, trích xuất IOC, sinh checklist điều tra, gợi ý câu hỏi cần xác minh và tạo bản nháp báo cáo. Hệ thống không thay analyst ra quyết định, mà giúp chuẩn hóa quy trình và giảm thời gian viết báo cáo.

1. Problem Statement

Trong SOC Tier 1, nhiều alert cần các bước giống nhau: xác định IP/domain/hash, kiểm tra reputation, đọc log liên quan, hỏi “có impact không?”, rồi viết kết luận. Nếu không có template, báo cáo dễ thiếu thông tin hoặc mỗi người viết một kiểu. Project này tập trung tự động hóa phần hỗ trợ, nhưng vẫn yêu cầu analyst xác nhận trước khi kết luận.

3. Core Features

IOC Extraction

Tự động nhận diện IP, domain, URL, hash, username, hostname từ alert text hoặc log mẫu.

Context Builder

Tạo bảng context gồm source, destination, timestamp, rule name, severity và asset role nếu có.

Investigation Checklist

Sinh câu hỏi cần kiểm tra: endpoint có process lạ không, DNS có domain liên quan không, firewall có block không.

Report Draft

Xuất bản nháp gồm summary, evidence, assessment, recommended action và limitation.

5. Sample Output Template

Alert Summary:
- Alert name: Suspicious outbound connection
- Source host: WIN-CLIENT01
- Destination IOC: 44.x.x.x
- First seen: 2026-04-xx 10:15
- Initial severity: Medium

Analyst Checklist:
[ ] Check endpoint process tree around alert time
[ ] Check DNS/proxy logs for related domain
[ ] Check firewall action: allowed or blocked
[ ] Check whether destination IP is repeated by other hosts
[ ] Decide: False Positive / Benign True Positive / True Positive / Escalate

6. Findings

  • Assistant hữu ích nhất ở bước chuẩn hóa câu hỏi điều tra và format báo cáo.
  • Không nên dùng AI để tự động đóng incident nếu thiếu evidence từ log thật.
  • Cần kết hợp parser/rule-based extraction trước, LLM chỉ hỗ trợ diễn giải và trình bày.
  • Phù hợp cho portfolio vì thể hiện tư duy SOC + automation + reporting.

7. Improvement Plan

  • Thêm connector đọc CSV/JSON export từ SIEM.
  • Thêm mapping MITRE ATT&CK theo rule name hoặc kỹ thuật liên quan.
  • Thêm risk scoring dựa trên asset criticality, reputation, frequency và action allowed/blocked.
  • Xuất report dạng Markdown/PDF để đính kèm vào case.
