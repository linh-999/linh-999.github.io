Executive Summary

Project này tập trung vào hướng phát triển hiện đại của SOC: dùng automation/AI để giảm thời gian đọc alert và chuẩn hóa report. AI không thay analyst kết luận; nó hỗ trợ tóm tắt field, gợi ý pivot, tạo checklist và draft báo cáo để analyst kiểm tra lại.

Objective & Scope

  • Parse alert field thành summary dễ đọc.
  • Extract IOC gồm IP, domain, URL, hash, username, hostname.
  • Gợi ý triage checklist theo alert type.
  • Draft report theo format SOC nhưng yêu cầu analyst validate trước khi dùng.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • SIEM xuất alert JSON hoặc analyst copy alert fields.
  • Python parser chuẩn hóa field và extract IOC.
  • Enrichment module gọi reputation/local lookup hoặc mapping internal context.
  • LLM tạo triage questions và report draft.
  • Analyst kiểm tra, chỉnh sửa và đưa verdict cuối.
Evidence artifacts: topology, dashboard, rule detail, raw event và command output được quản lý như bằng chứng kỹ thuật của case.

Detection Logic / Use Cases

  • Không tạo detection mới; tập trung enrichment và report automation.
  • Mapping alert type sang checklist: phishing, auth failure, endpoint suspicious process, network IDS.
  • Highlight missing evidence để analyst biết cần thu thập thêm.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Input alert raw fields.
  2. Extract IOC và chuẩn hóa timestamp/source/destination/user/host.
  3. Tạo checklist investigation theo type.
  4. Sinh draft summary, evidence table và recommendation.
  5. Analyst review để loại bỏ hallucination và cập nhật verdict thật.

Key Findings

  • AI hữu ích cho chuẩn hóa report và checklist.
  • Không nên dùng AI làm nguồn kết luận duy nhất.
  • Chất lượng output phụ thuộc chất lượng alert fields và prompt.

Recommendations

  • Luôn lưu raw evidence cạnh AI summary.
  • Dùng template cố định để giảm hallucination.
  • Không đưa dữ liệu nhạy cảm lên dịch vụ ngoài nếu chưa được phép.
  • Gắn nhãn confidence và missing evidence trong output.

Limitations & Next Improvements

  • Tích hợp local rule-based parser.
  • Thêm export markdown report.
  • Xây bộ test alert mẫu để đánh giá chất lượng draft.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
