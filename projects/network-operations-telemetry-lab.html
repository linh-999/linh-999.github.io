01 · Executive Summary

Executive Summary

Nghiên cứu cách vận hành mạng theo hướng telemetry: thu thập trạng thái interface, CPU/memory, link status, routing neighbor và traffic counters để phát hiện bất thường sớm hơn so với kiểm tra thủ công. Project kết nối định hướng NOC với SOC bằng cách đưa network health vào bối cảnh security monitoring.

Security operations value: Thể hiện khả năng nhìn hệ thống mạng như nguồn dữ liệu vận hành và bảo mật, không chỉ cấu hình thiết bị.
02 · Scope

Objective & Scope

  • Theo dõi interface status, utilization và error counters.
  • Ghi nhận bất thường như link flap, high bandwidth, packet drop.
  • Tạo alert rõ impact và recommended troubleshooting.
  • Liên hệ network telemetry với security event khi traffic tăng bất thường.

05 · Triage process

Investigation Workflow

  1. Xác định metric bất thường và thiết bị/port liên quan.
  2. Kiểm tra thời điểm thay đổi có trùng maintenance không.
  3. Đối chiếu traffic/security event nếu utilization tăng bất thường.
  4. Khoanh vùng physical, L2, L3 hoặc policy issue.
  5. Ghi recommendation cho NOC/SOC: monitor, replace cable, tune threshold hoặc investigate traffic source.

07 · Findings

Key Findings

  • Telemetry giúp phát hiện vấn đề vận hành trước khi người dùng báo lỗi.
  • High utilization cần được phân biệt giữa backup hợp lệ và traffic bất thường.
  • Network health là context quan trọng khi phân tích security alert.
08 · Response

Recommendations

  • Định nghĩa baseline cho core link và access link.
  • Tách alert vận hành và alert bảo mật nhưng cho phép correlation.
  • Chuẩn hóa dashboard theo device, interface, severity và service impact.
09 · Continuous improvement

Limitations & Next Improvements

  • Thử collector bằng Python và dashboard đơn giản.
  • Tích hợp log thiết bị vào SIEM để correlation.
  • Tạo runbook cho link flap và high utilization.
