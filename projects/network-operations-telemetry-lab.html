Executive Summary

Project mở rộng portfolio từ SOC sang vận hành mạng. Nội dung tập trung vào cách thu thập trạng thái thiết bị, phát hiện bất thường về interface/latency/loss, tự động hóa kiểm tra và tạo report để NOC/SOC có dữ liệu vận hành trước khi sự cố ảnh hưởng người dùng.

Objective & Scope

  • Mô tả mô hình giám sát Core IP/Transmission ở mức học thuật/lab.
  • So sánh polling truyền thống và telemetry push-based.
  • Đề xuất use case: interface flapping, high error rate, latency spike, routing change.
  • Viết hướng phát triển automation bằng Python.

Architecture / Log Flow

Alert / Event Source→Log Collection→SIEM / Dashboard→Analyst Triage→Verdict & Response
  • Network device sinh metric/log về interface, route và health.
  • Collector nhận dữ liệu định kỳ hoặc streaming.
  • Storage/indexer lưu metric theo time-series.
  • Dashboard hiển thị trạng thái và trend.
  • Automation script tạo health-check report hoặc cảnh báo.
Chèn ảnh lab tại đây: topology, dashboard, rule, event detail hoặc command output.

Detection Logic / Use Cases

  • Interface flap: trạng thái link đổi liên tục trong thời gian ngắn.
  • High error/discard: counter lỗi tăng bất thường.
  • Latency/loss spike: ảnh hưởng chất lượng dịch vụ.
  • Routing change: thay đổi route/neighbor cần kiểm tra.
# analyst-note
Detection should define: objective → data source → condition → expected evidence → false-positive notes → response action.

Investigation Workflow

  1. Xác định metric cần giám sát và ngưỡng cảnh báo.
  2. Thu thập dữ liệu mẫu từ thiết bị hoặc lab.
  3. Tạo dashboard trend theo thời gian.
  4. Viết checklist khi có alert: link, port, route, neighbor, traffic.
  5. Đề xuất action: monitor, open ticket, escalate transmission/core team.

Key Findings

  • Dữ liệu vận hành giúp SOC/NOC phân biệt sự cố mạng và sự cố bảo mật.
  • Automation giảm thời gian kiểm tra thủ công lặp lại.
  • Telemetry có giá trị cao khi cần phát hiện bất thường gần real-time.

Recommendations

  • Chuẩn hóa checklist health-check theo thiết bị/dịch vụ.
  • Lưu baseline để phát hiện deviation.
  • Tạo báo cáo định kỳ cho link/device critical.

Limitations & Next Improvements

  • Thử gNMI/Netconf/YANG ở lab nâng cao.
  • Tích hợp alert với Slack/email/ticket.
  • Kết hợp telemetry với security event trong SIEM.

Limitation được ghi rõ để tránh phóng đại kết quả lab. Khi có ảnh/log thật, phần này nên được cập nhật bằng evidence cụ thể.
